# 推送到 GitHub 的步骤

## 方案 1: 使用 Personal Access Token（推荐）

### 步骤 1: 创建 GitHub Token

1. 访问：https://github.com/settings/tokens
2. 点击 "Generate new token" → "Generate new token (classic)"
3. 设置：
   - Note: `CRMMVP Push Access`
   - Expiration: 选择合适的时间（如 90 天）
   - 勾选 `repo` 权限（完整仓库访问权限）
4. 点击 "Generate token"
5. **立即复制 token**（只显示一次！）

### 步骤 2: 使用 Token 推送

在命令行运行：

```bash
git remote set-url origin https://YOUR_TOKEN@github.com/handtalking/crmmvp.git
git push -u origin main
```

将 `YOUR_TOKEN` 替换为你刚才复制的 token。

## 方案 2: 使用 SSH（如果已配置 SSH 密钥）

```bash
git remote set-url origin git@github.com:handtalking/crmmvp.git
git push -u origin main
```

## 方案 3: 使用 GitHub CLI

```bash
gh auth login
git push -u origin main
```

## 方案 4: 在 GitHub 网页上手动上传

如果上述方法都不行：

1. 访问：https://github.com/handtalking/crmmvp
2. 点击 "uploading an existing file"
3. 将项目文件拖拽上传

## 注意事项

- ✅ `.env.local` 文件已被 `.gitignore` 排除，不会上传
- ✅ `node_modules` 不会被上传
- ⚠️ 确保你有 `handtalking/crmmvp` 仓库的写入权限
